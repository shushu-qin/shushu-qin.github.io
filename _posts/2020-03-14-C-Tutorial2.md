## 1 宏定义

**生成可执行文件的流程**

一个项目可以通过编译、链接形成一个可执行文件。在这个过程中，每个源文件都会单独编译，编译成一个目标文件（.o，.obj...），系统把这些.o文件进行链接，最终形成一个可执行文件。

**编译干了什么事？**

- 笼统的说：词法、语法分析、目标文件生成、优化。。。
- 拆开来看
  - 预处理
  - 编译：词法、语法分析、目标文件生成、优化，产生一些临时文件
  - 汇编：产生.o文件

其中的预处理是指：在源程序.cpp中加入一些特殊代码，这些特殊代码有一些特殊能力，提供一种特殊功能。

编译系统会先对这些特殊代码做处理，这个叫做”预处理“。处理结果和源代码一起进行编译、汇编。

**c语言提供三种预处理功能**

- 宏定义
- 文件包含
- 条件编译

这三种功能也是在程序中写代码来实现，只不过这些代码比较特殊，都是以#开头

### 1.1 不带参数的宏定义

```c
// #define 标识符 字符串
#define PI 3.1415926   // 不带参数的宏定义：用PI代替”3.1415926”这个字符串。程序源码中我们写的是PI，在编译预处理是，所有在该#define之后的PI都将被替换

```

好处：

- 用一个简单的名字代替长字符串，所以这个标识符也被成为宏名，在预编译时将宏名替换成字符串的过程叫做“宏展开”
- 增加了修改的方便性，也提高了程序的可移植性

说明：

- 宏名一般用大写字母

- 宏定义不是C语句，不必加分号，否则连分号一起替换

- #define命令出现在程序中函数的外面，从定义到本源文件末尾有效，不能跨文件使用

  可以把所用的define写在一个公共文件中

- 一般写在整个文件最上面

- 可以用#undef命令终止宏定义的作用域

```c
#define PI 3.1415926 
void fun1()
{
  ...
}
#undef PI
void fun2()
{
  ...
}
```

- 可在宏中使用已定义的宏
- 字符串内的字符不做宏替换

```C
char stmp[100] = "DSPI";
```

### 1.2 带参数的宏定义

#define 宏名(参数表) 字符串

```c
#define S(a,b) a*b
S(2,3)  //把2，3分别代替宏定义中的形式参数a,b ==》 2*3
S(1+5,2)  // 1+5*2
  				// 如果字符串中的字符不是参数字符，则保留
//解决方法：在形参外加个括号
```



```c
#define S(a,b) a*b
S(2,3)  // (a,b) a*b 宏名和形参括号间不能加空格
  
//宏替换复杂语句
#define MAX(x,y) (x)>(y)?(x):(y)
//宏替换多行语句
#define MACROTEST do {\
printf("test\n");\
} while(0);
```

宏定义和函数的区别：

- 函数调用是先求出实参表达式的值，然后代入形参，而带参数的宏只进行简单的字符替换
- 函数调用是在程序运行是处理，宏展开是在编译时进行，展开时不分配内存，也没有返回值这种说法，也没有值传递这种说法
- 宏的参数没有类型
- 使用宏次数多时，源代码变长，而函数调用不会
- 宏替换只占用编译时间，不占用运行时间，而函数调用占的是运行时间

## 2 文件包含和条件编译

### 2.1 文件包含

#include 文件名

常用于.h文件。我们常常把宏定义、函数说明、其他include命令、以及其他一些全局变量的外部申明等等放在头文件中。

### 2.2 条件编译

```
#ifdef 标识符
	程序段1
#else
	程序段
#endif
```

当标识符定义过，则对程序段1进行编译

```C
#define DEBUG
#ifdef DEBUG
	printf("This is debug version\n");
#else
	printf("This is release version\n");
#endif

#ifndef RELEASE
	printf("This is debug version\n");
#endif

#if PI
	printf("PI is defined\n");
#endif
```

条件编译的好处：

- 可以减少生成的目标文件长度
- 增加程序的可移植性

```c
#if _WIN32  // windows系统自己定义的
	// WaitForSingleObject()
	// windows 上专用
#elif __linux__
	// epoll()
#else

#endif
```
