## 1 文件概述

文件：看成字符序列（字符流）

根据数据组织形式，我们把文件分成过两种

- ASCII文件（文本文件）：每一个字节存一个ASCII码，这种文件一般你打开究竟能看懂里面的内容
- 二进制文件：把内存中的数据按照其在内存中的存储形式原样输出到磁盘上存放

对于计算机来讲，都是二进制文件，只有人类才把文件分成文本文件和二进制文件。

但是：你往文件中写入内容，你可以选择是是以文本形式写入，还是以二进制形式写入。你打开记事本写入，系统就会认为你是以文本形式写入。

或者，打开文件往外读也是一样。

```c
short int a; // 2 bytes
int ilen = sizeof(a);
a = 10000; //0x2710

// 低字节0x10存放在了低地址....D4
// 低字节0x10存放在了低地址....D5   vs里看10在27之前  小段存储
```

大端存储（低的存高地址里，高的存在低地址里） 和 小端存储（相反）

## 2 文件的打开、关闭

### 2.1 文件的打开

文件在读或者写之前必须要打开，读写结束后必须关闭，否则会造成文件内容不完整或资源泄露。

```c
// fopen: 文件打开函数
FILE *fp;
// fp = fopen(文件名，使用文件的方式)
// 文件名和使用文件的方式都是字符串 “”
fp = fopen("a.txt","r")  //只读打开a.txt，fp现在就指向了咱们打开的a.txt文件

```

`fp`是指向结构`FILE`的指针变量（结构体指针）。

通过fopen语句告诉了系统三个信息

- 需要打开的文件名
- 使用文件的方式
- 让那个指针变量指向被打开的文件

文件有个位置指针，fgetc自动往后移动一个字节。

|      |                                                |
| ---- | ---------------------------------------------- |
| r    | 以文本方式打开文件                             |
| r+   |                                                |
| rb+  |                                                |
| rt+  |                                                |
| w    | 如果file存在，则清空，若不存在，则创建该文件。 |
| w+   |                                                |
| a    |                                                |
| a+   |                                                |
| wb   |                                                |

fopen若失败，则返回`NULL`。

```c
if（fp==NULL)
{
   // 文件打开失败
}
```

### 2.2 文件的关闭 fclose

只有fopen成功的文件才需要关闭。

关闭文件的原因：

- 1）释放文件占用的内存单元，要养成资源用完及时释放的好习惯
- 2）防止往文件中写内容时文件内容写入不全：系统写文件时，先写到缓冲区，等缓存满了，才写入磁盘。关闭文件的动作会触发系统把缓冲区的数据立即写入磁盘。

```c
// fclose(文件指针)
if(fp!==NULL)
{
  fclose(fp);
}
```

## 3 文件的读写

fputc和fget函数

```c
// fputc: 该函数吧一个字符写到磁盘文件上去，一般形式为
// fputc(ch, fp) //ch就是这个字符，如果fputc失败，会返回EOF，如果成功，则返回那个字符的ASCII码值。
// EOF: end of file  // #define EOF -1
FILE *fp;
fp = fopen("test.txt","w");
if(fp == NULL)
{
	printf("File to open the file\n");
}
else
{
  char reco = fputc('a',fp);
  if(reco == EOF)
  {
    // fail to write
	}
  reco = fputc('d',fp);
  reco = fputc('d',fp);
  fclose(fp);
}

// fgetc: 从指定文件读入一个字符
// char reco = fget(fp);
// 如果fputc失败，会返回EOF，如果成功，则返回读入的字符
FILE *fp;  // 文件刚打开，文件位置位于文件头部
fp = fopen("test.txt","r");
if(fp == NULL)
{
	printf("File to open the file\n");
}
else
{
  char reco = fgetc(fp);
  while(!(feof(fp)))  // 
  {
    putchar(reco); // 往屏幕上输出一个字符
	}
  printf("\n");
  fclose(fp);
}

// feof(fp) 文件是否结束
// 如果文件结束，返回1真， 否则返回0（假）

```

### 5 结构体与二进制文件

### 5.1 将结构体写入二进制文件

```c
#pragma pack(1) // 1字节对齐结构
struct stu
{
  char name[30];  // 30  // 2
  int age;        // 4   // 4
  double score;   // 8
  // 不要用指针
  // vs编译器为了提高程序运行效率，可能会在int里填4个字节 => 48
  // 在gcc编译器上（4字节对齐），可能这个结构体就占44个字节
  // 所以在linux上读会错位
}
#pragma pack()

int main()
{
  // fwrite: 用于向文件中写入数据
  // fwrite(buffer, size, count, fp)
  // buffer: 指针/地址
  // size: 要写入文件的字节数
  // count: 要写入多少个size字节的数据项
  // fp: 文件指针
  // 返回值: 若失败，返回0，否则返回count值
  struct stu student[2]; // 2个元素的结构体数组
  strcp(student[0].name,"张三abc");
  student[0].age = 21;
  student[0].score = 90.1f;
  
  strcp(student[0].name,"李四def");
  student[0].age = 19;
  student[0].score = 81.2f;
  
  FILE *fp;
  fp = fopen("structfile.bin","wb");
  if(fp == NULL)
  {
    prinf("打开文件失败");
  }
  else
  {
    int result = fwrite(student, sizeof(struct stu), 2, fp);
    fclose(fp);
  }
}
```

注意事项

- 往文件里写的结构体中不要出现指针类型变量

- 结构体内存对齐问题，和编译器有关

  解决办法：

  	-  不要跨平台
  	-  无论什么平台，都1字节对齐（不用对齐）
