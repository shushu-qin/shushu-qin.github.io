## 2. 移动构造函数和移动赋值运算符的概念

c++11中引入的

引入原因：

- 效率问题：拷贝的成本是很高的

说明：

- A移动给B，那么A对象就不能再被我们使用了
- 移动：并不是把内存中的数据从一个地址倒腾到另外一个地址，只是所有者变更

```c++
//拷贝构造函数
//Time::Time(const Time &tmpTime){...} //const左值引用&

//移动构造函数
//Time::Time(const Time &&tmpTime){...} //右值引用&&
//如果有其他参数，则必须有默认值，这点同拷贝构造函数一样
```

移动构造函数和移动赋值运算符应该完成的功能：

- 资源的移动，斩断原对象和这段内存的关系
- 确保移动后原对象处于“一种即便被销毁也没有问题”的状态。B <- A
- 确保不再使用A，而是应该使用B

## 3. 移动构造函数演示

```c++
class B
{
public:
  //默认构造函数
  B(): a(0)
  {
    cout << "B的构造函数执行了" << endl;
  }
  
  //拷贝构造函数
  B(const & t): a(t.a)
  {
     cout << "B的拷贝函数执行了" << endl;
  }
  //析构函数
  virtual ~B() //以后写子类的时候。。。
  {
    cout << "B的析构函数执行了" << endl;
  }
  
  int a;
}

class A
{
public:
  //默认构造函数
  A(): pb(new B())
  {
    cout << "A的构造函数执行了" << endl;
  }
  //拷贝构造函数
  A(const A &ta): pb(new B(*(ta.pb)))
  {
    cout << "A的拷贝构造函数执行了" << endl;
  }
  //移动构造函数
  //nonexcept声明和实现都得加，通知标准库我们这个移动构造函数不抛出任何异常
  //为了提高编译器工作效率
  //移动构造函数习惯性的要加nonexcept
  A(const A &&ta) nonexcept : pb(ta.pb) //原来对象a指向的内存pb，直接就让临时对象指向这段内存
  {
    //把a指向的内存打断
    ta.pb = nullptr;
    cout << "A的移动构造函数执行了" << endl;
  }
  //拷贝赋值运算符
  A& operator = (const A& src)
  {
    if (this == &src)
    {
      return *this;
    }
    delete pb;
    pb = new B(*(src.pb));
    cout << "A的拷贝赋值运算符执行了" << endl;
    return *this;
  }
  //移动赋值运算符
  A& operator = (const A&& src) nonexcept
  {
    if (this == &src)
    {
      return *this;
    }
    delete pb; //把自己的内存先干掉
    pb = src.pb; //对方的内存直接拿来
    src.pb = nullptr; //把对方的内存斩断
    cout << "A的移动赋值运算符执行了" << endl;
    return *this;
  }
  
  //析构函数
  virtual ~A() //以后写子类的时候。。。
  {
    delete pb;
    cout << "A的析构函数执行了" << endl;
  }
private:
  B *pb;
}

//表示这个函数只能在当前文件中被调用
static A getA()
{
  A a;
  return a; //临时对象，调用拷贝构造函数
  					//如果类A有移动构造函数，则调用移动构造函数，把a对象移动给了临时对象
}

int main()
{
  B *pb = new B(); //调用默认构造函数
  pb->a = 10;
  B *pb2 = new B(*pb); //调用拷贝构造函数
  
  delete pb, pb2;
  
  A a = getA(); //调用了1次构造函数，1次拷贝构造函数，两次析构函数，从而程序结束
                //增加移动构造函数后，调用一次构造函数，一次移动构造函数，两次析构函数，从而程序结束
  A b(a); //a是左值，所以调用拷贝构造函数
  //如果a在后续代码中不被使用的话，直接用移动构造函数
  A b(std::move(a)); //调用移动构造函数
  //A &&b(std::move(a)); //这里没有建立新对象，根本没有调用移动构造函数
  											 //等同于把对象a的名字改为b
  
  A &&c = getA(); //getA返回的临时对象被c接管了
  
  return 0;
}
```
移动赋值运算符

```c++
A a = getA();
A a2;
a2 = std::move(a); //调用移动赋值运算符
```

## 4. 合成的移动操作

某些条件下，编译器能合成移动构造函数和移动赋值运算符。

- 有自己的拷贝构造函数，自己的拷贝赋值运算符，或自己的析构，这三者之一，那么编译器就不会为它合成移动构造函数和移动赋值运算符。所以有一些类是没有移动构造函数和移动赋值运算符的

- 如果我没没有自己的移动构造函数和移动赋值运算符，那么系统会调用我没自己写的拷贝构造函数和拷贝赋值运算符

- 只有一个类没定义任何版本的拷贝构造函数和拷贝赋值运算符，且类的每个非静态函数都可以移动，编译器才会为它合成移动构造函数和移动赋值运算符。

  什么叫成员可以移动

  - 内置类型可以移动

  - 类类型成员，则这个类要有对应的移动相关的函数就可以移动

    ```c++
    struct TC
    {
      int i; //内置类型可以移动
      std::string a; //有自己的移动操作
    }
    
    TC a;
    a.i = 100;
    a.s = "I love china";
    const char *p = a.s.c_str();
    TC b = std::move(a); //导致TC移动构造函数的执行（系统构造的）
    const char *q = b.s.c_str();
    //p, q地址不一样，这种移动不是真正的移动，由于string对象本身的特性
    ```


## 5. 总结

- 尽量给类添加移动构造函数和移动赋值运算符
- 不抛出异常的移动构造函数和移动赋值运算符都加上noexcept
- 移动构造函数里要人为置空被移动对象
- 没有移动会调用拷贝代替
