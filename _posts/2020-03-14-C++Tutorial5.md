## 1 左值和右值

```c++
int i = 10; //i 对象：一块内存区域
//左值：能用在赋值语句左边的东西，它能够代表一个地址

//右值：不能作为左值的值就是右值
//结论：
//c++的一条表达式，要么是左值要么是右值，不可能两者都不是
//左值有时可以被当做右值使用
i = i + 1; //i是个左值，虽然它出现在了等号右边
					 //i用在等号右边时，我们说i有一种右值属性（不是右值）
					 //i出现在等号左边，用的是i代表的内存中的地址，我们说i有一种左值属性
```

一个左值，它可能同时具有左值属性和右值属性。

用到左值的运算符有哪些：

- 赋值运算符

- 整个赋值语句的结果仍然是左值

  ```c++
  int a;
  printf("%d\n", a = 4);
  (a = 4) = 8;
  ```

- 取地址

  ```c++
  int a = 5;
  &a;
  ```

- string vector 下标[]

- 迭代器

  ```c++
  vector<int>::iterator iter;
  iter++; iter--;
  ```

左值表达式就是左值

右值表达式就是右值

## 2 引用分类

```c++
int value = 10;
int &rvalue = value;
rvalue = 100;
```

分类：

- 左值引用

- const引用，也是左值引用

  ```c++
  const int &rvalue = value;
  //rvalue = 18; //fail
  ```

- 右值引用

  ```c++
  int &&rvalue = 3; //绑定到一个常量
  rvalue = 5;
  ```

### 2.1 左值引用

没有空引用的说法，所以左值引用初始化的时候就绑定左值。

```c++
int a = 1;
int &b{a};
//int &c; //fail
//int &c = 1; //fail
const int &c = 1;
//<==>
//int tmpvalue = 1;
//const int &c = tmpvalue;
```

### 2.2 右值引用

系统希望用右值引用来绑定一些即将销毁的或者临时的对象。

右值引用也是引用。大家理解成一个对象的名字。

```c++
int &&rhv = 3;
rhv = 9;
int value = 10;
//int &&rhv = value //fail 右值引用绑定不到左值引用上
string str{"I love China"};
string &rstr{str};
//string &rstr{"I love China"} //fail 左值引用不能绑定临时变量
const string &rstr{"I love China"} //const引用不但可以绑定到右值，还可以执行string的隐式转换，并将得到的值放到string的临时变量中
string &&rstr{"I love China"};
```

总结：

- 返回左值引用的函数，连同赋值，下标，解引用和前置递增递减(--i, ++i)，都是返回左值表达式的例子，我们可以将左值引用绑定到这类表达式的结果上。
- 返回非引用类型的函数，连同算数，关系，位，以及后置递增运算符(i--)，都生成右值。不能将一个左值引用绑定到这类表达式上，但是我们可以将一个const的左值引用或者一个右值引用绑定到这类表达式上。

++i 左值表达式。直接给变量i 减一，然后返回i。因为i是变量，所以可以被赋值。

```c++
int i = 100;
(++i) = 199;
```

i++右值表达式。先产生一个临时变量，记录i的值，再给i+1，返回给临时变量。

```c++
(i++)  = 100; //fail
```

```c++
int i = 1;
int &&r1 = i++; 
// i += 5;
//int &r2 = i++; //fail
int &r3 = ++i;
i += 5;
//int &r4 = ++i; //右值不能绑定左值表达式
int &r5 = r1; //r1是左值
```

重点强调：

- r1虽然是右值引用，但这个名字本身是个左值，看成一个变量，因为它在等号左边待着

- 所有变量都看成左值

- 任何函数里的形参都是左值。void f(int i, int &&r1): r1是右值引用，但它本身是左值
- 临时对象都是右值

### 2.3 右值引用的目的

- 是c++11引入的新概念，用&&表示，代表一个新数据类型，引入新数据类型肯定有目的
- 把拷贝对象变成移动对象提高程序运行效率
- 移动对象如何发生
  - 移动构造函数、移动赋值运算符需要&&
  - 拷贝构造函数、拷贝赋值运算符需要&

## 5 std::move函数

c++11标准库里的新函数。

实际上这个函数根本没有做移动的操作。

std::move的能力只有一个：把一个左值强制转换成一个右值，带来的结果就是我一个右值可以绑上去了。

```c++
int i = 10;
//int &&ri1  = i;
int &&ri1 = std::move(i);
i = 20;
ri1 = 15; //ri2代表了i

int &&ri2 = 100;
//int &&ri3 = ri2;
int &&ri3 = std::move(ri2);
ri2 = 68;
ri3 = 10;
```

```c++
string str = "I love China";
const char *p = str.c_str();
string def = std::move(str); //是string本身的移动构造函数把str里的内容清空了，而不是std::move干的。string重新开辟内存，然后把"I love China"存到自己的内存中了
const char *q = def.c_str(); //p和q指向的内存地址不一样
```

```c++
string str = "I love China";
std::move(str); //str不为空

string &&def = std::move(str); //不会触发string的移动构造函数
//str = "abc"; //def也随之改变
def = "adasd"; //str也随之改变

```

系统建议：用完std::move后，move的参数最好不要再使用了。
