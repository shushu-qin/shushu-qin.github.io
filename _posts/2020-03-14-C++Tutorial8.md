## 1 隐式类型转换

系统自动进行，不需要程序开发人员介入。

```c++
int m = 1+45.6;
```

## 2 显示类型转换（强制类型转换）

```c++
//int k = 5%3.2 //语法错误
// int k = 5 % (int)3.2; //OK  c语言风格的强制类型转换
// int k = 5 % int(3.2); //OK  c语言风格/函数风格的强制类型转换

```

c++强制类型转换，提供更丰富的含义和功能，更好的类型检查机制

- static_cast
- dynamic_cast
- const_cast
- reinterpret_cast

也称为命名的强制类型转换。

通用形式

​	强制类型转换名<type>(express);

### 2.1 static_cast

大家理解成正常转化，编译时就会进行类型转换的检查。代码中要保证转换的安全性和正确性，和C语言中强制类型转换感觉差不多。c风格的强制类型转换以及编译器能够进行的隐式类型转换，都可以用static_cast类显示完成。

可用于：

- 相关类型的转换

  ```c++
  double f = 100.23f;
  int i = (int) f;
  int i = static_cast<int>(f);
  ```

- 子类和父类的转换

  ```c++
  class A {};
  class B : publica A{};
  B b;
  A a = static_cast<A> b; //把子类转成父类
  //B b = static_cast<B> a; //fail
  ```

- void *与其他类型指针之间的转换，void *：无类型指针，可以指向任何的指针类型（万能指针）

  ```c++
  int i = 10;
  int *p = &i;
  //void *q = p;
  //int *qq = q;
  void *q = static_cast<void *>(p);
  int *qq = static_cast<int *>q;
  ```

不可用于：

- 一般不能用于指针类型的转换，比如int *转double *等

### 2.2 dynamic_cast

运行是类型识别和检查。主要用来父类型和子类型之间转换用的（父类型指针指向子类型对象，然后用dynamic_cast转）。

### 2.3 const_cast

取出指针或者引用的const属性，该转换能够将const性质转换掉。

编译时就会进行类型转换。

```c++
const int ai = 90;
// int ai2 = const_cast<int>(ai); //fail ai既不是指针也不是引用
const int *pai = &ai;
int *pai2 = const_cast<int *>(pai);
//*pai2 = 100; //这种写值行为属于一种未定义行为，大家不要这么干，会产生无法预料的后果
cout << ai; //90
//cout << *pai; //120
```

### 2.4 reinterpret_cast

编译的时候就会进行类型转换的检查。

将操作室内容解释为另一种不同的类型，能把操作数的类型都转了。

处理无关类型的转换。就等于想怎么转就怎么转。

常用于：

- 将一个整型（地址）转换成指针，按照转换后的内容重新解释内存中的内容；

- 也可以从一个指针类型转换成一个整型

  ```c++
  //int i = 10;
  //int *pi = &i;
  //int *p2 = reinterpret_cast<int *>(&i);
  //char *p2 = reinterpret_cast<char *>(&i);
  
  int i = 10;
  int *pi = &i;
  void *pvoid = reinterpret_cast<void *>(pi);
  int *pi2 = reinterpret_cast<int *>(pvoid);
  
  ```



## 3 总结

- 强制类型转换不建议使用，它会一直编译器报错
- 学习的目的：认识这些类型转换符，方便大家阅读别人的代码
- 资料说：reinterpret_cast危险，使用const_cast以为这设计缺陷
- 如果是在需要类型转换，不要再使用C语言类型转换，而是c++类型的
- 一般static_cast和reinterpret_cast就能够很好的取代C语言类型转换
- 虽然reinterpret_cast被认为是危险的类型转换，怎么转编译器都不报错。只要好好用，其实还是有其实际用途的
